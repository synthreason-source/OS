Storage-First OS

A minimal operating system whose primary services are persistent, addressable multidimensional arrays with on-storage computation primitives — including an optimised matrix multiplier and a programmable ASCII windowing subsystem.

§ 2 — High-Level Design

Architecture

RPC / API Layer
NumPy-style client library translating array calls into short RPCs. Lazy tile pulls, copy-on-write views, and broadcasting rules.
userspace
Execution Primitives
Storage-resident signed kernels: blocked GEMM, elementwise ufuncs, reductions. Double-buffered I/O overlaps reads with compute.
RX-signed
ASCII UI Service
Character-mode window manager with programmable handlers, keyboard event queue, and custom dot-cursor pattern renderer.
kernel
Access Control
Per-object capability tokens enforcing R / W / RX / TX flags. Every syscall is checked, logged with object ID, user token, and timestamp.
kernel
Storage Layer
Typed object store: UUID, shape tuple, dtype, layout ('C'/'F'), tile block-index, CRC/version manifests, optional compressed block pointers.
persistent
⊞

Array-First Semantics

Every persistent resource is a typed ndarray. Shape, dtype, layout, and CRC are first-class metadata stored alongside tile data.

⚡

Near-Storage Compute

Kernels execute in-place on storage, streaming tile blocks to minimise host DRAM pressure and reduce PCIe I/O bandwidth.

🔒

Capability Security

Tokens are issued per-handle per-mode. RX requires code signing. TX requires owner ACL. Every operation is audited.

⌨

Minimal ASCII UI

Windows backed by 2D char/attribute arrays. Cursor patterns stored as small array objects. Keyboard events in ring buffers.

§ 6 — Capability Model

R / W / RX / TX Access Modes

Mode Name Description Permitted operations
R Read Read-only access to object tiles. No metadata or data changes. read_tile open(r) slice view query meta
W Write Create new versions or modify tiles. Requires explicit write consent from owner. write_tile new version copy-on-write
RX Read-Execute Register and run storage-resident kernels. Requires code signing. Restricted by default. call_kernel run GEMM ufunc exec set_cursor
TX Transmit Export signed object snapshots to external nodes or removable media. Requires ACL or owner consent. export snapshot sign manifest send to network

§ Appendix — API Sketch

API Reference

Storage API

storage.create_array(shape, dtype, order='C', tile=None) → handle
Create a persistent ndarray object. Mirrors numpy dtype/order semantics. Returns an opaque handle for subsequent operations.
storage.open(handle, mode='r'|'w'|'rx'|'tx') → token
Acquire a capability token. The kernel validates this token on every syscall. Tokens are session-scoped and audited.
storage.read_tile(handle, tile_index) → bytes
Read raw tile bytes; decompresses on-the-fly if the tile is stored with run-length or quadtree encoding.
storage.write_tile(handle, tile_index, data, token)
Atomic tile write using write-to-temp + rename semantics. Requires W token. Checksums are updated on commit.
storage.call_kernel(name, handles, args, token) → job_id
Invoke a signed storage-resident kernel (e.g. 'gemm'). Requires RX token. Returns a job ID for async polling.

UI API

ui.create_window(handle_or_shape, pos, size) → window_id
Create a text window backed by a storage array of char/attribute pairs. Can bind to an existing array handle.
ui.set_cursor(window_id, cursor_handle)
Assign a dot cursor pattern (3×3 or 5×5 uint8 array) to a window. RX token required on cursor_handle.
ui.read_keys(window_id) → bytes | events
Drain the keyboard event queue for a window. Supports synchronous poll and event-driven callback modes.

Example usage flow

# 1. Allocate matrices A = storage.create_array((4096, 4096), dtype='float64') B = storage.create_array((4096, 4096), dtype='float64') C = storage.create_array((4096, 4096), dtype='float64') # 2. Acquire RX token and invoke GEMM kernel tok = storage.open(C, mode='rx') job = storage.call_kernel('gemm', (A, B, C), {'alpha': 1.0}, tok) # kernel streams tiles, computes in blocks, commits C atomically # 3. Open an ASCII window bound to C's tile region win = ui.create_window(C, pos=(0, 0), size=(80, 24)) cur = storage.create_array((3, 3), dtype='uint8') ui.set_cursor(win, cur) events = ui.read_keys(win)
